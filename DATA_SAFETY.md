# Google Play 数据安全声明填写指南

在 Google Play Console 发布应用时，需要填写"数据安全"（Data Safety）表单。以下是 Agentra 应用的填写建议：

## 数据收集与共享概览

| 问题 | 回答 |
|------|------|
| 应用是否收集或共享任何必需的用户数据类型？ | **是** |
| 所有收集的用户数据是否已加密传输？ | **是**（所有 API 调用均通过 HTTPS） |
| 是否提供用户请求删除数据的途径？ | **是**（卸载应用即删除所有数据；也可在设置中清除） |

## 数据类型明细

### 收集的数据

| 数据类型 | 是否收集 | 是否共享 | 说明 |
|----------|----------|----------|------|
| 个人信息（姓名、邮箱等） | 否 | 否 | - |
| 财务信息 | 否 | 否 | - |
| 位置信息 | 否 | 否 | - |
| 网络浏览历史 | 否 | 否 | - |
| 设备或其他标识符 | 否 | 否 | - |
| **应用活动（用户生成内容）** | **是** | **是** | 用户在会议中输入的讨论主题和上下文 |

### "应用活动"数据详情

- **收集目的**：应用功能（发送给 LLM API 以生成 AI 响应）
- **是否共享给第三方**：是 — 发送给用户自行配置的 LLM 服务提供商
- **共享的第三方**：Anthropic、DeepSeek、Moonshot、智谱（取决于用户配置）
- **是否可选**：否（这是应用核心功能）
- **加密传输**：是（HTTPS/TLS）
- **是否可请求删除**：是（卸载即删除本地数据；第三方 API 的数据保留请参考各服务商政策）

### API 密钥（注意事项）

API 密钥属于用户提供的凭证，**仅存储在本地设备**，不属于 Google Play 数据安全声明中需要披露的"个人数据"类别。但建议在应用描述和隐私政策中明确说明。

## Play Console 填写步骤

1. 进入 Play Console → 应用 → **政策** → **应用内容** → **数据安全**
2. 回答"概览"部分的三个问题（参考上表）
3. 在"数据类型"部分，勾选 **应用活动 → 其他用户生成的内容**
4. 填写数据用途为"应用功能"
5. 标注数据会与第三方共享（LLM 服务商）
6. 确认数据加密传输
7. 确认提供数据删除机制
8. 提交并等待审核
